# Задача 1: Архитектура целевого состояния системы

## Содержимое раздела

1. **c4_containers_diagram.puml** - C4 диаграмма контейнеров целевой архитектуры в формате PlantUML
2. **problems_analysis.md** - Исследование проблемных зон существующей архитектуры с ранжированием

## Описание концепции

Разработана целевая архитектура для организации «Будущее 2.0» на годовой горизонт, устраняющая ключевые ограничения текущей монолитной системы.

### Основные архитектурные подходы:

1. **Предметно-ориентированная архитектура** - выделение клинической, ИИ, финансовой и аналитической областей
2. **Современный стек данных** - переход с устаревшего SQL Server на актуальные решения (Data Lake, потоковая обработка, OLAP)
3. **Веб-портал для пользователей** - единая точка входа к аналитическим возможностям для бизнес-аналитиков
4. **Микросервисный подход** - автономное развитие и горизонтальное масштабирование модулей
5. **Событийно-ориентированная интеграция** - несинхронный обмен информацией между предметными областями

### Ранжирование проблем:

Применены методики MoSCoW и матрица Эйзенхауэра для оценки критичности ограничений и определения последовательности их устранения.
