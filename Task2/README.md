# Задача 2: Предметно-ориентированная архитектура и потоки данных

## Содержимое раздела

1. **data_flow_diagram.puml** - Data Flow Diagram (DFD) в формате PlantUML, демонстрирующая потоки данных между предметными областями
2. **domain_separation_rationale.md** - Подробное обоснование логики разделения на предметные области

## Описание концепции

### Определенные предметные области:
1. **Клиническая область** - управление клиническими данными и лечебными процессами
2. **Область искусственного интеллекта** - обучение моделей и анализ клинических данных  
3. **Финансовая область** - финансовые продукты и управление расчетными операциями
4. **Аналитическая область** - объединение данных и формирование отчетности
5. **Интеграционный уровень** - взаимодействие между областями и внешними системами

### Основные принципы разделения:
- **Бизнес-ориентированность** - каждая область соответствует бизнес-функции
- **Автономность данных** - каждая область владеет своими данными
- **Слабая связанность** - взаимодействие через четко определенные API
- **Автономное развитие** - возможность эволюции областей без влияния на другие

### Выгоды архитектуры:
- Автономное развитие бизнес-направлений
- Повышенная защищенность и соответствие требованиям
- Горизонтальное масштабирование и производительность
- Ускоренное подключение новых направлений
- Улучшенная устойчивость к сбоям

## Потоки данных

DFD демонстрирует:
- Потоки данных от внешних источников (врачи, клиенты, партнеры)
- Межобластные взаимодействия
- Хранилища данных каждой области
- Миграционные потоки из устаревшей системы
- Аналитические потоки для отчетности
